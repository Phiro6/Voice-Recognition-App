# Voice-to-AppleScript AI Agent (no03)

マイクから入力した音声（日本語）をオンデバイスAIで高精度に文字起こしし、その内容をローカルLLMへ連携してmacOS操作用のAppleScriptを自動生成・実行する、ハンズフリーのMacコントロール・アシスタントアプリです。

---

## 🚀 主な機能・特徴

- **オンデバイス音声文字起こし**: OpenAIのWhisperモデル（WhisperKit）を組み込み、外部サーバーに音声を送ることなく完全にローカル（Mac内）で高速・高精度な日本語認識を行います。
- **ローカルLLM連携（Ollama）**: 通信のプライバシーを確保しつつ、軽量で強力なオープンソースLLM（Gemma等）に指示（プロンプト）を送り、正確なAppleScriptコードを生成します。
- **自律型スクリプト実行**: AIが生成したAppleScriptコードを自動で抽出し、システムネイティブな実行環境（`NSAppleScript`）を用いてMac上の各種アプリケーション（Excel、Safariなど）を実際に最前面で起動・操作します。
- **完全ローカル動作**: 音声データやプロンプトを外部クラウドに一切送信しないため、セキュリティや機密性の高いビジネス環境でも安心して利用可能です。

## 🛠️ 採用技術・スタック

- **Language**: Swift 5.10+
- **Frameworks**:
  - `Cocoa` / `AppKit` (macOS UIツールキット)
  - `AVFoundation` / `AVAudioEngine` (高度な音声入力およびリアルタイムフォーマット変換)
- **AI / ML**:
  - `WhisperKit` (Apple Siliconに最適化されたCoreMLベースのWhisper実行環境)
  - `Ollama API` (ローカルLLM推論サーバーとのHTTP/JSON通信)
- **Asynchronous Programming**: Swift Concurrency (`Task`, `async/await`), `DispatchQueue` (GCD)

## 📊 システムアーキテクチャ・データフロー

本アプリは、マイクの入力からプログラムの実行まで、以下の5つのフェーズを一本道のパイプラインで処理しています。

1. **音声キャプチャ & リアルタイム変換 (`AVFoundation`)**
   - マイク入力（環境依存: 44.1kHz等）を、WhisperKitが求める厳格なフォーマット（16kHz / Mono / 32bit Float PCM）に `AVAudioConverter` を用いてリアルタイムでダウンサンプリングしながら配列に蓄積。
2. **オンデバイス音声文字起こし (`WhisperKit`)**
   - 蓄積された音声波形データを非同期（`Task` / `async await`）でWhisperKitに投入し、日本語テキスト（例：「Excelを開いて」）を抽出。
3. **プロンプトエンジニアリング & AI推論 (`Ollama`)**
   - 抽出したテキストを「解説文を省き、純粋なAppleScriptのコードブロックのみを出力せよ」という厳格な制約プロンプトに加工。`URLSession` を用いてローカルのOllamaサーバー（ポート11434）へHTTP POSTリクエスト。
4. **コード抽出 (`NSRegularExpression`)**
   - AIから返却されたテキストから、正規表現パターン（````applescript ... ````）を用いて純粋なスクリプト文字列のみをパース。
5. **システム実行 (`NSAppleScript`)**
   - 抽出したAppleScriptを、macOSのAPIを通じてカーネルレベルで安全に実行。

## 💡 技術的なこだわりとトラブルシューティング（面接アピールポイント）

### 1. 高級なリアルタイム音声変換（AVAudioConverterの制御）
マイク入力を直接AIに渡すことはできません。本プロジェクトでは、`installTap` を使ってバッファサイズ4096単位で音声をリアルタイムキャッチし、標的サンプリングレートとの比率から厳密に必要バッファ容量（`frameCapacity`）を動的計算し、`UnsafeBufferPointer` を用いて安全にSwiftの `[Float]` 配列へブリッジする低レイヤーの音声処理ロジックを実装しました。

### 2. 2回目以降の連続録音エラーの完全回避
初期開発時、録音停止に伴い音声エンジン自体（`AVAudioEngine`）をストップさせていたため、2回目の録音開始時にハードウェアの再初期化ラグが発生し、アプリがクラッシュ・拒否されるバグに直面しました。
* **改善**: エンジンは常時バックグラウンドで起動（スタンバイ）状態を維持させ、録音の開始・停止は音声監視用の「タップの着脱（`removeTap` / `installTap`）」のみで制御する設計へと変更。これにより、タイムラグやエラーが一切ないシームレスな連続録音環境を実現しました。

### 3. macOS App Sandbox（セキュリティ制限）の克服
開発の最終段階で、スクリプト実行時に **エラーコード `-600 (Application isn't running)`** が発生し、外部アプリが操作できない課題が発生しました。原因を深く調査した結果、macOSのセキュリティ機能である「アプリのサンドボックス化」により、他アプリへのアクセス命令（AppleEvents）がOSレベルで遮断されていることを特定しました。
* **解決**: Xcodeの `Signing & Capabilities` 設定において、適切なアクセス権限（Hardware: Audio Input の許可、およびAppleEvents通信の許可、またはサンドボックスの解除）を明示的に構成。OSのセキュリティ仕様に則った正確な権限管理のもとでアプリ連携を実現しました。

### 4. スレッド管理の徹底（UIフリーズの防止）
重い処理（WhisperKitのロード、音声認識、URLSessionによるHTTP通信）は、すべてSwift Concurrencyやバックグラウンドスレッドで非同期実行。画面を書き換える処理（`NSTextView`へのテキスト反映やボタンの活性化）を行う直前には、必ず **`DispatchQueue.main.async`** を用いて確実にメインスレッドへ処理を引き戻し、UIのフリーズや不正なスレッドアクセスを100%防ぐスレッドセーフな設計を徹底しました。

## 🛠️ セットアップ & 実行方法

### 前提条件
- macOS 14.0+ (Apple Silicon搭載のMacを推奨)
- Xcode 15.0+
- [Ollama](https://ollama.com/) がローカルでインストールされ、起動していること
  - 使用モデル: `gemma4`（プロンプトに応じて、インストールされている他のモデルにコード内で変更可能です）
